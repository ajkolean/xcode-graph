# Layout Components

Application layout and navigation components that provide the structural foundation.

## 🎯 Layout Structure

```
┌─────────────────────────────────────────────────┐
│ Header (top navigation)                         │
├──────────┬──────────────────────────┬───────────┤
│          │                          │           │
│ Sidebar  │   Graph Content Area     │  Right    │
│          │   (GraphTab)             │  Sidebar  │
│          │                          │           │
│          │   - GraphVisualization   │ - Filters │
│          │   - Toolbar              │ - Details │
│          │                          │           │
└──────────┴──────────────────────────┴───────────┘
```

## 🧩 Components

### Header
Top navigation bar with:
- Organization breadcrumbs
- Logo and branding
- Documentation link
- User profile avatar

**Position**: Fixed top, full width

### Sidebar (Left)
Primary navigation with tabs:
- Overview
- Builds
- Test Runs
- Module Cache
- Analytics
- Settings

**Features**:
- Tab switching
- Active tab highlighting
- Icon-based navigation

### Toolbar
Graph controls overlay:
- Zoom controls (in/out/reset)
- Node and edge count statistics
- Animation toggle

**Position**: Absolute, top-left of graph area

### Right Sidebar
Context-sensitive details panel that shows:
- **Filters view** (default)
- **Node details** (when node selected)
- **Cluster details** (when cluster selected)

**Features**:
- Collapsible/expandable
- State-based content switching
- Managed by Zag state machine

### RightSidebarHeader
Header for the right sidebar with:
- Title (changes based on view)
- Collapse toggle button

### CollapsedSidebar
Compact vertical icon bar shown when right sidebar is collapsed:
- Filter section icons with active badges
- Node/Edge count stats

### PlaceholderTab
Placeholder content for non-graph tabs (Builds, Test Runs, etc.)

## 🎨 Layout Principles

### Responsive Behavior
- Sidebar: 240px fixed width
- Right Sidebar: 320px expanded, 56px collapsed
- Graph area: Flexible, fills remaining space

### Z-Index Layers
1. Background (graph canvas)
2. Graph content (nodes, edges)
3. Overlays (toolbar, controls)
4. Panels (sidebars, modals)

### Transitions
All layout changes use smooth transitions:
- Sidebar collapse/expand: 200ms ease
- Panel switching: Fade in/out
- Hover states: 150ms

## 📱 Future Considerations

While not currently implemented, the layout structure supports:
- Responsive breakpoints
- Mobile-optimized views
- Panel resizing
- Multi-column layouts
