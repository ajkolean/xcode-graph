## Form Composition Principles

### Accessibility First
- Every input has an associated `<label>`
- Use `aria-describedby` for help text
- Mark required fields with `aria-required`
- Provide clear error messages with `role="alert"`

### Visual Hierarchy
- Group related fields together
- Use consistent spacing (var(--spacing-4) between fields)
- Align labels and inputs properly
- Provide visual feedback for states (focus, error, disabled)

### User Experience
- Show validation errors after user interaction
- Provide helpful placeholder text
- Use appropriate input types (email, tel, etc.)
- Include clear submit/cancel actions

## Basic Form Pattern

### Simple Contact Form

```tsx
import { LitInput } from '../components-lit/wrappers/Input';
import { LitLabel } from '../components-lit/wrappers/Label';
import { LitButton } from '../components-lit/wrappers/Button';

function ContactForm() {
  return (
    <form style={{ display: 'flex', flexDirection: 'column', gap: 'var(--spacing-4)', maxWidth: '400px' }}>
      {/* Name Field */}
      <div>
        <LitLabel htmlFor="name">Name</LitLabel>
        <LitInput
          id="name"
          type="text"
          placeholder="John Doe"
          required
          aria-required="true"
        />
      </div>

      {/* Email Field */}
      <div>
        <LitLabel htmlFor="email">Email</LitLabel>
        <LitInput
          id="email"
          type="email"
          placeholder="john@example.com"
          required
          aria-required="true"
        />
      </div>

      {/* Message Field */}
      <div>
        <LitLabel htmlFor="message">Message</LitLabel>
        <LitTextarea
          id="message"
          placeholder="Your message..."
          rows={4}
        />
      </div>

      {/* Submit */}
      <LitButton type="submit" variant="default">
        Send Message
      </LitButton>
    </form>
  );
}
```

**Key Features**:
- Consistent spacing with design tokens
- Proper label associations
- Required field indicators
- Semantic HTML structure

## Form with Validation

### Client-Side Validation Pattern

```tsx
import { useState } from 'react';

function ValidatedForm() {
  const [errors, setErrors] = useState({});

  const handleSubmit = (e) => {
    e.preventDefault();
    const formData = new FormData(e.target);
    const errors = {};

    // Validate email
    const email = formData.get('email');
    if (!email?.includes('@')) {
      errors.email = 'Please enter a valid email address';
    }

    if (Object.keys(errors).length > 0) {
      setErrors(errors);
      return;
    }

    // Submit form...
  };

  return (
    <form onSubmit={handleSubmit} style={{ display: 'flex', flexDirection: 'column', gap: 'var(--spacing-4)' }}>
      <div>
        <LitLabel htmlFor="email">Email</LitLabel>
        <LitInput
          id="email"
          name="email"
          type="email"
          aria-invalid={errors.email ? 'true' : 'false'}
          aria-describedby={errors.email ? 'email-error' : undefined}
        />
        {errors.email && (
          <div
            id="email-error"
            role="alert"
            style={{
              marginTop: 'var(--spacing-2)',
              fontSize: 'var(--font-sizes-sm)',
              color: 'var(--colors-destructive)',
            }}
          >
            {errors.email}
          </div>
        )}
      </div>

      <LitButton type="submit">Submit</LitButton>
    </form>
  );
}
```

**Accessibility**:
- `aria-invalid` indicates validation state
- `aria-describedby` links error message to input
- `role="alert"` announces errors to screen readers
- Visual styling differentiates error state

## Multi-Step Form

### Wizard Pattern

```tsx
function MultiStepForm() {
  const [step, setStep] = useState(1);

  return (
    <div style={{ maxWidth: '500px' }}>
      {/* Progress Indicator */}
      <div style={{ marginBottom: 'var(--spacing-6)' }}>
        <div style={{ display: 'flex', gap: 'var(--spacing-2)', marginBottom: 'var(--spacing-2)' }}>
          {[1, 2, 3].map((i) => (
            <div
              key={i}
              style={{
                flex: 1,
                height: '4px',
                background: i <= step ? 'var(--colors-primary)' : 'var(--colors-muted)',
                borderRadius: 'var(--radii-full)',
              }}
            />
          ))}
        </div>
        <div style={{ fontSize: 'var(--font-sizes-sm)', opacity: 0.7 }}>
          Step {step} of 3
        </div>
      </div>

      {/* Step Content */}
      {step === 1 && (
        <div style={{ display: 'flex', flexDirection: 'column', gap: 'var(--spacing-4)' }}>
          <h3>Personal Information</h3>
          <LitInput placeholder="First Name" />
          <LitInput placeholder="Last Name" />
        </div>
      )}

      {step === 2 && (
        <div style={{ display: 'flex', flexDirection: 'column', gap: 'var(--spacing-4)' }}>
          <h3>Contact Details</h3>
          <LitInput type="email" placeholder="Email" />
          <LitInput type="tel" placeholder="Phone" />
        </div>
      )}

      {step === 3 && (
        <div style={{ display: 'flex', flexDirection: 'column', gap: 'var(--spacing-4)' }}>
          <h3>Preferences</h3>
          <LitCheckbox>Subscribe to newsletter</LitCheckbox>
          <LitCheckbox>Enable notifications</LitCheckbox>
        </div>
      )}

      {/* Navigation */}
      <div style={{ display: 'flex', gap: 'var(--spacing-2)', marginTop: 'var(--spacing-6)' }}>
        {step > 1 && (
          <LitButton variant="outline" onClick={() => setStep(step - 1)}>
            Back
          </LitButton>
        )}
        <LitButton onClick={() => step < 3 ? setStep(step + 1) : handleSubmit()}>
          {step === 3 ? 'Submit' : 'Next'}
        </LitButton>
      </div>
    </div>
  );
}
```

**Pattern Features**:
- Visual progress indicator
- Step-by-step navigation
- Back/Next buttons
- Final submit action

## Form with Help Text

### Descriptive Form Pattern

```tsx
<form style={{ display: 'flex', flexDirection: 'column', gap: 'var(--spacing-4)', maxWidth: '400px' }}>
  <div>
    <LitLabel htmlFor="username">Username</LitLabel>
    <LitInput
      id="username"
      aria-describedby="username-help"
    />
    <div
      id="username-help"
      style={{
        marginTop: 'var(--spacing-2)',
        fontSize: 'var(--font-sizes-sm)',
        color: 'var(--colors-muted-foreground)',
      }}
    >
      Choose a unique username between 3-20 characters
    </div>
  </div>

  <div>
    <LitLabel htmlFor="password">
      Password <span style={{ color: 'var(--colors-destructive)' }}>*</span>
    </LitLabel>
    <LitInput
      id="password"
      type="password"
      aria-describedby="password-help"
      required
      aria-required="true"
    />
    <div
      id="password-help"
      style={{
        marginTop: 'var(--spacing-2)',
        fontSize: 'var(--font-sizes-sm)',
        color: 'var(--colors-muted-foreground)',
      }}
    >
      Must be at least 8 characters with a mix of letters and numbers
    </div>
  </div>

  <LitButton type="submit">Create Account</LitButton>
</form>
```

**Best Practices**:
- Help text linked via `aria-describedby`
- Required fields marked visually and semantically
- Clear password requirements
- Appropriate spacing between fields

## Checkbox Group Pattern

```tsx
<form>
  <fieldset style={{ border: 'none', padding: 0 }}>
    <legend style={{
      fontSize: 'var(--font-sizes-h4)',
      fontWeight: 'var(--font-weights-semibold)',
      marginBottom: 'var(--spacing-3)',
    }}>
      Select your interests
    </legend>

    <div style={{ display: 'flex', flexDirection: 'column', gap: 'var(--spacing-3)' }}>
      <div style={{ display: 'flex', alignItems: 'center', gap: 'var(--spacing-2)' }}>
        <LitCheckbox id="interest-tech" />
        <LitLabel htmlFor="interest-tech">Technology</LitLabel>
      </div>

      <div style={{ display: 'flex', alignItems: 'center', gap: 'var(--spacing-2)' }}>
        <LitCheckbox id="interest-design" />
        <LitLabel htmlFor="interest-design">Design</LitLabel>
      </div>

      <div style={{ display: 'flex', alignItems: 'center', gap: 'var(--spacing-2)' }}>
        <LitCheckbox id="interest-business" />
        <LitLabel htmlFor="interest-business">Business</LitLabel>
      </div>
    </div>
  </fieldset>
</form>
```

**Accessibility**:
- `<fieldset>` groups related inputs
- `<legend>` provides group label
- Individual labels for each checkbox
- Keyboard navigation works automatically

## Best Practices

### ✅ Do's

**Use Semantic HTML**
```tsx
// ✅ Proper semantic structure
<form>
  <fieldset>
    <legend>Group Label</legend>
    {/* inputs */}
  </fieldset>
</form>
```

**Label Every Input**
```tsx
// ✅ Explicit labels
<LitLabel htmlFor="email">Email</LitLabel>
<LitInput id="email" />
```

**Provide Help Text**
```tsx
// ✅ Descriptive help
<LitInput aria-describedby="help" />
<div id="help">Helper text</div>
```

**Handle States**
```tsx
// ✅ Clear state indicators
<LitInput
  disabled={loading}
  aria-invalid={hasError}
  aria-describedby={hasError ? 'error' : 'help'}
/>
```

### ❌ Don'ts

**Don't Skip Labels**
```tsx
// ❌ No label (inaccessible)
<LitInput placeholder="Email" />

// ✅ Always include label
<LitLabel htmlFor="email">Email</LitLabel>
<LitInput id="email" />
```

**Don't Use Placeholder as Label**
```tsx
// ❌ Placeholder disappears on focus
<LitInput placeholder="Enter email" />

// ✅ Persistent label + helpful placeholder
<LitLabel>Email</LitLabel>
<LitInput placeholder="john@example.com" />
```
