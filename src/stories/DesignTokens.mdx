## Overview

Design tokens are stored as **CSS custom properties** in `src/styles/tokens.css` and are available in both:
- **Light DOM** (`:root`) - For React components
- **Shadow DOM** (`:host`) - For Lit components

This dual-context approach ensures maximum compatibility and consistent styling across frameworks.

**Total Tokens**: 75+ design tokens across 7 categories

## Animation

### Durations

**4 duration tokens** for consistent timing:

| Token | Value | Use Case |
|-------|-------|----------|
| `--durations-fast` | 150ms | Hover effects, highlights |
| `--durations-normal` | 200ms | Standard transitions |
| `--durations-slow` | 300ms | Panel expansions, reveals |
| `--durations-slower` | 500ms | Page transitions, complex animations |

### Easings

**4 easing functions** for natural motion:

| Token | Value | Use Case |
|-------|-------|----------|
| `--easings-default` | cubic-bezier(0.4, 0, 0.2, 1) | General purpose |
| `--easings-in` | cubic-bezier(0.4, 0, 1, 1) | Elements entering |
| `--easings-out` | cubic-bezier(0, 0, 0.2, 1) | Elements exiting |
| `--easings-in-out` | cubic-bezier(0.4, 0, 0.2, 1) | Bidirectional |

**Usage**:
```css
transition: all var(--durations-normal) var(--easings-default);
```

## Using Design Tokens

### In React Components

```tsx
// Using inline styles
<div style={{
  padding: 'var(--spacing-md)',
  background: 'var(--colors-card)',
  borderRadius: 'var(--radii-md)',
  boxShadow: 'var(--shadows-sm)',
}}>
  Content
</div>

// Using className (if using utility classes)
<div className="p-md bg-card rounded-md shadow-sm">
  Content
</div>
```

### In Lit Components

Tokens are available in Shadow DOM via `:host`:

```typescript
@customElement('graph-card')
export class GraphCard extends LitElement {
  static styles = css`
    :host {
      display: block;
      padding: var(--spacing-md);
      background: var(--colors-card);
      border-radius: var(--radii-md);
      box-shadow: var(--shadows-sm);
    }
  `;
}
```

### In CSS Files

```css
.my-component {
  /* Colors */
  color: var(--colors-foreground);
  background: var(--colors-background);

  /* Spacing */
  padding: var(--spacing-4);
  gap: var(--spacing-2);

  /* Typography */
  font-family: var(--fonts-body);
  font-size: var(--font-sizes-base);
  font-weight: var(--font-weights-medium);
  line-height: var(--line-heights-normal);

  /* Borders */
  border-radius: var(--radii-md);
  border: 1px solid var(--colors-border);

  /* Effects */
  box-shadow: var(--shadows-sm);
  transition: all var(--durations-normal) var(--easings-default);
}
```

## Best Practices

### ✅ Do's

**Use Semantic Tokens First**
```css
/* ✅ Semantic names are self-documenting */
padding: var(--spacing-md);
color: var(--colors-primary);

/* ⚠️ Numeric tokens are less clear */
padding: var(--spacing-4);
```

**Maintain Consistency**
```tsx
// ✅ Use tokens everywhere
<Button style={{ padding: 'var(--spacing-md)' }} />

// ❌ Don't mix tokens and hard-coded values
<Button style={{ padding: '16px' }} />
```

**Compose Tokens**
```css
/* ✅ Combine multiple tokens */
.card {
  padding: var(--spacing-lg);
  background: var(--colors-card);
  border-radius: var(--radii-md);
  box-shadow: var(--shadows-md);
}
```

### ❌ Don'ts

**Don't Hard-Code Values**
```css
/* ❌ Hard-coded values break design system */
color: #6f2cff;
padding: 24px;
font-size: 14px;

/* ✅ Use tokens instead */
color: var(--colors-primary);
padding: var(--spacing-6);
font-size: var(--font-sizes-base);
```

**Don't Create Custom Tokens Without Review**
```css
/* ❌ Ad-hoc tokens fragment the system */
--my-custom-spacing: 15px;

/* ✅ Use existing tokens or propose new ones */
padding: var(--spacing-4); /* 16px, close enough */
```

## Extending the Design System

### Adding New Tokens

To add new design tokens:

1. **Add to `src/styles/tokens.css`**:
   ```css
   :where(:root, :host) {
     --colors-my-new-color: rgba(100, 200, 50, 1);
   }
   ```

2. **Document in this file** (DesignTokens.mdx)

3. **Update TypeScript types** (if applicable)

4. **Use in components**:
   ```tsx
   <div style={{ color: 'var(--colors-my-new-color)' }}>
   ```

5. **Test in both React and Lit** to ensure compatibility

### Token Naming Convention

Follow the pattern: `--[category]-[name]`

Examples:
- `--colors-primary`
- `--spacing-md`
- `--font-sizes-h1`
- `--radii-lg`
- `--durations-fast`
