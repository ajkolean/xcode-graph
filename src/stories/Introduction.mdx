## Quick Start

### Installation

```bash
# Install dependencies
pnpm install

# Run Storybook in development mode
pnpm storybook

# Build Storybook for production
pnpm build-storybook
```

### Using Components

Import components from the library:

```tsx
// React Components
import { Button } from './components/button';
import { Badge } from './components/badge';

// Lit Web Components (with React wrappers)
import { Button as LitButton } from './components-lit/wrappers/Button';
import { GraphInput } from './components/graph-input';

function MyApp() {
  return (
    <div>
      <Button variant="default">React Button</Button>
      <LitButton variant="secondary">Lit Button</LitButton>
    </div>
  );
}
```

## Architecture

Our component library features a unique dual-implementation approach:

### **React Components**
- Built with **React 19** and **Panda CSS**
- Type-safe with **TypeScript**
- Direct integration with React applications
- Optimal for React-native projects

### **Lit Web Components**
- Standards-based **Web Components** using **Lit 3**
- **Shadow DOM** encapsulation for style isolation
- Framework-agnostic - works with React, Vue, Angular, or vanilla JS
- Smaller bundle sizes for production

### **Parity Testing**
- Side-by-side comparison ensures visual and functional consistency
- Both implementations share the same design tokens
- Comprehensive test coverage using Vitest and Playwright

## Design System

### Panda CSS
Our styling foundation leverages [Panda CSS](https://panda-css.com/) for:
- **Type-safe styling** - Autocomplete and type checking for CSS properties
- **Design tokens** - Centralized color, spacing, and typography scales
- **Zero-runtime** - Styles extracted at build time
- **Responsive utilities** - Mobile-first responsive design patterns

### Shadow DOM
Lit components use Shadow DOM to provide:
- **Style encapsulation** - No CSS leakage or conflicts
- **Composition** - Slot-based content projection
- **Standards compliance** - Built on web platform APIs

## Best Practices

### Accessibility First
All components are designed with accessibility in mind:
- **WCAG 2.1 Level AA** compliance
- **Keyboard navigation** support
- **Screen reader** compatibility
- **Focus management** and indicators

See our [Accessibility Guide](/story/foundations-accessibility--keyboard-navigation) for demonstrations.

### Shadow DOM Testing
When testing Lit components, use our Shadow DOM utilities:

```tsx
import { within as withinShadow } from 'shadow-dom-testing-library';

// In your tests
const canvas = withinShadow(canvasElement);
const button = await canvas.findByShadowRole('button');
```

Learn more in our [Testing Guide](/story/introduction-testing-guide).

### Parity Testing
When adding new components:
1. Implement in both React and Lit
2. Create parity story showing both side-by-side
3. Ensure identical visual appearance and behavior
4. Add interaction tests for both implementations

See our [Parity Testing Guide](/story/introduction-parity-testing) for details.
